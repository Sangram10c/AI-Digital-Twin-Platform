## Purpose

Define naming, architecture patterns, and engineering conventions for the monorepo.

## Scope

Cross-cutting coding standards for frontend, backend, and shared packages.

## Content

## Architecture Patterns

- **Clean Architecture**: Separate concerns into layers
- **SOLID Principles**: Single responsibility, open-closed, etc.
- **Feature-First**: Group by feature, not by type
- **Repository Pattern**: Abstract data access
- **Service Layer**: Business logic in services
- **DTO Pattern**: Validate and transform input/output
